# Service Name Review

Date: 2026-05-09
Status: Decided

## Decision

The service name is `つぎたく`.

Primary copy:

- Name: `つぎたく`
- Tagline: `前回のログを、次回の卓へ。`
- Descriptor: `セッションログから、記憶と次回準備をつくるGM作業台`

## Why This Works

- Japanese-first and easy to say at the table.
- Short, hiragana, and memorable without sounding like generic SaaS.
- Directly evokes `次の卓`, which matches the core workflow: turn the previous session log into next-session prep.
- Broader than a log tool because it can cover memory, prep, handouts, and future hosted workflows.
- Softer and more approachable than English fantasy-coded names.

## Risks

- Casual tone may feel lighter than the product's feature depth.
- Search and trademark availability are not verified here.
- Because `つぎたく` is a coined shorthand, the first screen should always pair it with the tagline or descriptor.

## Rejected Alternatives

| Name | Reason |
| --- | --- |
| Loreline | Memorable, but too abstract and English/fantasy-coded for a Japan-first launch. |
| Tableline | Clearer than Loreline, but still English-coded and less catchy in Japanese conversation. |
| ログつむぎ | Good functional image, but narrower and more descriptive than brandable. |
| たくしるべ | Strong meaning, but calmer and less immediately catchy. |
| よみつぎ | Elegant, but the TRPG/table use case needs too much explanation. |

## Follow-Up Checks

- Trademark search.
- Domain and social handle search.
- Search-result ambiguity for `つぎたく`.
- Visual logo check at small sizes using hiragana.
